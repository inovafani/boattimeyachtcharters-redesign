Website Edit Guide

How to update text, images, and backgrounds on your website
without needing a developer.

Prepared April 2026

1 How your website is built

Think of your website like a magazine. Every section you see on screen is its own file saved in a folder called components/. To change something, you open that file, update the words or image URL, save, and the website updates instantly.

The golden rule: Only ever touch text between >words here< or URLs inside quote marks. Never touch the surrounding code characters like < > { }.

The three tools you need

1
VS Code — the editor where you open and change files. It's already installed.
2
Terminal (inside VS Code) — you type npm run dev here to start the live preview.
3
Chrome at localhost:3000 — this shows you a live preview that updates every time you save a file.

3 How to update text (copy)

Text in your website lives between > and < symbols. Here is an example from the whale section:

<h2 className="whale-headline">
  Humpbacks,<br />
  <em>at close range.</em>
</h2>

To change it, simply replace the words:

<h2 className="whale-headline">
  Whales,<br />
  <em>up close.</em>
</h2>
Tip: Text wrapped in <em> tags shows in italic gold on screen. Keep those tags if you want the gold italic effect.
Do NOT touch: Anything that starts with className=, style=, ref=, gsap, useRef, or useEffect. These control layout and animations.

Step by step

1
Open VS Code and open the project folder.
2
Find the file using the table in Section 2. Click it in the left sidebar.
3
Use Find — press Ctrl + F, type a few words from the text you want to change. It highlights exactly where it is.
4
Change the words. Only touch the actual text, not the code around it.
5
Save with Ctrl + S. The browser at localhost:3000 updates automatically.

4 How to update a photo

Photos inside the page use an <img> tag with a src= attribute. That src is just a URL pointing to an image on the internet.

<img
  src="https://boattimeyachtcharters.com/.../EDI_3071-scaled.jpg"
  alt="Humpback whale breach from Sun Goddess"
/>

To swap the photo, replace the URL inside src="..." with your new image URL. Leave everything else untouched.

How to get your image URL:
Go to your WordPress dashboard at boattimeyachtcharters.com → Media Library → Upload your photo → Click the photo → Copy the "File URL" shown on the right side. Paste that URL into the src="...".

5 How to update a background image

Background images are set in two different places depending on the section.

Way 1 — Inside the component file (most sections)

Look for backgroundImage: inside a style={{ }} block. Example from the Hero section:

backgroundImage:
  'url(https://images.unsplash.com/photo-1567899378...)',

Replace the URL between url( and ) with your new image URL. Keep url( and ) exactly as they are.

Way 2 — Inside globals.css (whale section, final CTA)

Open app/globals.css. Press Ctrl + F and search for the section name (e.g. .whale-bg). You'll find:

.whale-bg {
  background-image: url('https://boattimeyachtcharters.com/.../EDI_3071.jpg');
  background-size: cover;
}

Replace the URL inside url('...') with your new image URL.

Best image sizes: For hero/full-width backgrounds use at least 1920px wide. For section backgrounds 1400px is fine. Keep file size under 500 KB for fast loading — use squoosh.app (free) to compress before uploading.

6 How to update the bottom info bar

The dark bar at the bottom of every page (showing sunset time, sea state, next whale etc) is controlled by components/TickerBar.tsx.

Open the file. Near the top you'll see this block:

const ITEMS = [
  { label: "Today's Sunset", value: getSunsetTime(), accent: true },
  { label: 'Sea State',     value: 'Calm · 0.4m' },
  { label: 'Next Whale',    value: 'Sat 08:30 · Sun Goddess' },
  { label: 'Availability',  value: '3 dates this week' },
];

Only change the text inside the value: quotes. Leave label: and everything else alone.

7 How to go live with the whale cam

When Kirby starts streaming on YouTube, open components/LiveStream.tsx. At the very top of the file you'll see:

// ── Update these 3 lines when going live ──────────────────
const IS_LIVE           = false;
const YOUTUBE_VIDEO_ID  = 'YOUR_VIDEO_ID_HERE';
const NEXT_STREAM_DATE  = 'Sat 28 Jun · 08:00 AEST';
1
Change false to true on the IS_LIVE line.
2
Get the YouTube video ID. When Kirby goes live, open YouTube Studio. The live stream URL will look like youtube.com/watch?v=abc123xyz. Copy the part after ?v= — that is the video ID.
3
Paste the video ID between the quotes on the YOUTUBE_VIDEO_ID line. Replace YOUR_VIDEO_ID_HERE.
4
Save the file. The section automatically switches from the offline poster to the live YouTube player.
5
When the stream ends, change true back to false and update NEXT_STREAM_DATE to the next planned date.

8 How to publish your changes live

Your website is hosted on Vercel. Every time you push your changes to GitHub, Vercel automatically rebuilds and publishes the site within about 2 minutes.

1
Open the Terminal inside VS Code. Go to Terminal → New Terminal in the menu bar.
2
Type these three commands, pressing Enter after each one:
git add .
git commit -m "updated whale section text"
git push
3
Wait about 2 minutes, then check your live website. Vercel does the rest automatically.
What those commands mean:
git add . — packages all your changed files
git commit -m "..." — labels what you changed (put any description in the quotes)
git push — sends it to GitHub, which triggers Vercel to publish
